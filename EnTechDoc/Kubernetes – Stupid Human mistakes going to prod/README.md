正如任何新的平台、应用或者服务，学习什么是不该做的机会是永远存在的，当他被以正确的方式使用，很多人将为此收益。因此，事不宜迟，这里我们列举了在生产过程使用kubernetes的一些建议

* 灾难恢复测试应该有计划的被执行

* 不要让版本发生真正快速的、次要的改变

* 不要让任何人做

* Kube-DNS资源是重要且必要的

* 要充分沟通

* ETCD……最好不要管他

* init 0 不是一个"重启"服务的好命令
